## Notes

- **Tool**: [GitHub Copilot](https://github.com/features/copilot) (in-editor agent mode) was used throughout the entire project.
- **Design** (`assist`): The human owned all architectural decisions — choice of Blazor Server, Papra integration model, IMAP polling approach, EF Core + SQLite data layer, and the document processing pipeline shape. Copilot generated initial implementations that the human frequently redesigned or replaced when the architecture did not meet requirements.
- **Implementation** (`pair`): Both the human and Copilot acted on the codebase. The human wrote independent changes throughout, corrected generated code, and did not rely solely on Copilot output. Neither party dominated.
- **Testing** (`copilot`): The full test suite (114 tests covering services, repositories, models, background services, and Blazor UI components via bUnit) was generated by Copilot. The human identified coverage gaps and directed improvements.
- **Documentation** (`copilot`): README, inline comments, and release notes were written by Copilot based on human direction.
- **Review** (`hint`): The human performed all code review independently. Copilot surfaced passive in-editor suggestions (warnings, quick fixes) but did not actively participate in review decisions.
- **Deployment** (`pair`): CI/CD pipeline design was a collaborative effort. The human directed requirements and made key decisions (platform matrix, versioning strategy, release structure); Copilot wrote the YAML and iterated on fixes alongside the human.
